Este proyecto demuestra cómo SonarQube puede automatizar la revisión de calidad en el ciclo de vida del desarrollo mediante el análisis de dos implementaciones, Python y C#, de una misma herramienta interna: un asistente local de migración a SharePoint. El script escanea archivos locales, calcula métricas y genera planes de migración en CSV o JSON, todo ello sin dependencias cloud y ejecutable en cualquier máquina.

La versión Python (local_migration_assistant.py) incluye errores no fatales como clases duplicadas, violación del principio ISP, variables con mayúsculas inconsistentes, métodos duplicados, variables no modificadas, complejidad innecesaria, constantes mágicas, nombres pobres, código comentado y código muerto. La versión C# (LocalSharePointMigrationAssistant.cs) reproduce los mismos patrones adaptados al lenguaje, con namespaces duplicados, interfaces infladas, NotImplementedException, métodos idénticos y similares code smells.

Para garantizar la calidad, se ha definido un Quality Gate específico para el proyecto. Los umbrales no son arbitrarios: se ha fijado cero bugs y cero vulnerabilidades en código nuevo porque cualquier fallo funcional o de seguridad es inaceptable incluso en herramientas internas. La mantenibilidad debe ser rating A (deuda técnica inferior al 5%) para fomentar código limpio desde el inicio. La cobertura de tests se ha establecido en un mínimo del 60%, un objetivo realista que incentiva la escritura de pruebas sin bloquear el desarrollo. La duplicación no puede superar el 3% para desincentivar el copia y pega. Finalmente, solo se exigen estas condiciones cuando el cambio supera las 20 líneas de código nuevo, evitando penalizar correcciones menores.

SonarQube detecta todos los problemas introducidos y, con este Quality Gate, el proyecto no supera el análisis hasta que se corrijan. Así se demuestra que la herramienta puede actuar como guardián automático de la calidad en el SDLC, tanto en Python como en C#, de forma justificada y adaptada al contexto del proyecto.
